  <name>Task 1: Create LLMProcessingProvider protocol and OllamaProcessingEngine</name>
  <files>Sources/OptionC/Processing/LLMProcessingProvider.swift, Sources/OptionC/Processing/OllamaProcessingEngine.swift</files>
  <action>
Create directory `Sources/OptionC/Processing/`.

Create `LLMProcessingProvider.swift`:
- `import Foundation`
- Define `protocol LLMProcessingProvider` with one method: `func process(_ text: String) async throws -> String`
- Doc comment: "Abstraction over any LLM post-processing backend. OllamaProcessingEngine conforms today. A future AnthropicProcessingEngine conforms without touching AppState."

Create `OllamaProcessingEngine.swift`:
- `import Foundation`
- Define `enum OllamaError: Error` with cases: `invalidResponse`, `httpError(Int)`, `emptyOutput`, `outputLengthSuspect`
- Define `final class OllamaProcessingEngine: LLMProcessingProvider` with `static let shared = OllamaProcessingEngine()`
- Private properties: `baseURL = URL(string: "http://localhost:11434")!`, `model: String`, `timeoutSeconds: TimeInterval`
- `init(model: String = "llama3.2", timeout: TimeInterval = 60)` — default timeout is 60s to handle cold-start (STATE.md concern)
- `func process(_ text: String) async throws -> String`:
  1. Build URL: `baseURL.appendingPathComponent("api/chat")`
  2. Create `OllamaChatRequest` with model, messages array (system message + user message), `stream: false`
  3. Create `URLRequest` with POST method, Content-Type application/json, encoded body, `timeoutIntervalForRequest = timeoutSeconds`
  4. Call `URLSession.shared.data(for: request)`
  5. Guard `HTTPURLResponse` and statusCode == 200, else throw `OllamaError.httpError(statusCode)` or `.invalidResponse`
  6. Decode `OllamaChatResponse`, extract `message.content`, trim whitespace
  7. Guard output is not empty, else throw `.emptyOutput`
  8. Output length guard: if `output.count > text.count * 3 && output.count > 200`, throw `.outputLengthSuspect` with NSLog warning
  9. Return output

- Placeholder system prompt (Phase 5 will tune): "You are a transcription cleanup engine. Fix punctuation and capitalisation. Return ONLY the cleaned text. No explanation, no preamble, no quotes."

- Codable models (private to file or internal):
  - `struct OllamaChatRequest: Encodable` — `model: String`, `messages: [OllamaMessage]`, `stream: Bool`
  - `struct OllamaMessage: Codable` — `role: String`, `content: String`
  - `struct OllamaChatResponse: Decodable` — `message: OllamaMessage`, `done: Bool`

IMPORTANT: Always set `stream: false` explicitly. Ollama streams by default and URLSession will receive fragmented JSON, breaking JSONDecoder. See Research Pitfall 1.
